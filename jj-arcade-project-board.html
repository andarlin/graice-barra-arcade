JJARC-001
Add cardFront to jj-arcade.json All memory cards showing GB logo placeholder instead of character images
To Do
Critical
5 min
jj-arcade.json

Description

The memory game card front face relies on window.JJ_ARCADE_CONFIG.cardFront to show the face-down logo. This property doesn't exist in jj-arcade.json, so cardFront is undefined — resulting in a broken image src. All 12 cards show a broken image or nothing on the front face.

Root Cause (memory.js line ~75)

src="${(window.JJ_ARCADE_CONFIG && window.JJ_ARCADE_CONFIG.cardFront) ? window.JJ_ARCADE_CONFIG.cardFront : ''}" // ↑ cardFront is never set → empty string → broken image

Fix — Add to jj-arcade.json (top level)

"cardFront": "/wp-content/themes/hello-elementor/assets/jj-arcade/cards/gb_3d_logo.png"

Acceptance Criteria

  • All 12 face-down cards show the GB logo
  • On match, card back reveals the correct character image
  • No broken image icons visible at any point
  • Verified in both desktop and mobile view

JJARC-002
Enqueue technique-match.js in PHP template Technique Match game completely broken — script never loaded by WordPress
To Do
Critical
5 min
template.php

Description

technique-match.js (5.9 KB, fully built) is never enqueued by the PHP template. When a user clicks Technique Match, the arcade loader polls for window.JJGames["technique-match"] and times out after 3 seconds, showing "Game module not found." The script simply needs to be registered alongside memory.js.

Fix — Add to template-jiujitsu-arcade.php

$tm_path = jj_arcade_asset_path('games/technique-match.js'); if (file_exists($tm_path)) { wp_enqueue_script( 'jj-arcade-technique-match', jj_arcade_asset_url('games/technique-match.js'), ['jj-arcade-js'], filemtime($tm_path), true ); }

Acceptance Criteria

  • Clicking "Technique Match" loads the game board
  • 12 cards render (6 name + 6 description cards)
  • Matching name → description marks both as matched
  • Victory screen shows after all 6 pairs matched
  • No "module not found" error in console

JJARC-003
Remove duplicate loadGame() function in jj-arcade.js Function defined twice — second definition silently overwrites the first
To Do
Medium
5 min
jj-arcade.js

Description

loadGame() is declared twice in jj-arcade.js. Both are currently identical so it doesn't break anything, but it's a maintenance risk — if someone edits one and not the other you get subtle bugs. The first definition (which wraps waitForModule) should be kept, and the duplicate removed.

Acceptance Criteria

  • Only one loadGame() definition exists in jj-arcade.js
  • The retained version uses waitForModule()
  • All games still load correctly after edit

belt-order.js

Description

Build the belt order drag-and-drop game. Player sees shuffled belt cards and must drag them into the correct GB Junior belt progression sequence. Show visual feedback on correct/incorrect placement. Register as window.JJGames["belt-order"].

Game Flow

  • Load belt data from dataUrl in game config
  • Shuffle belts and render as draggable cards
  • Player drags cards into a numbered drop zone (1–10)
  • On submit: highlight correct (green) / incorrect (red)
  • Show score and "Try again" button
  • Stretch: reveal correct order on failure

Technical Notes

  • Use HTML5 Drag & Drop API (no external libs)
  • Register: window.JJGames["belt-order"] = { mount }
  • Follow same mount(root, gameConfig) pattern as other games
  • Mobile: use touch events fallback or tap-to-select-and-place

Acceptance Criteria

  • All 10 belts displayed as draggable cards
  • Drag and drop works on desktop
  • Tap-to-place works on mobile
  • Score shown after submission
  • Restart resets the board
